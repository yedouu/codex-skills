# Research Report Template

## `.reference-research/README.md`

必须包含：
- 本次调研目的。
- 推荐阅读顺序。
- 每个文件的作用。
- 如何继续深挖。
- 如何让 Codex 根据调研结果实现功能。
- 如何更新调研。

## `research_brief.md`

必须包含：
- 用户原始需求。
- 需求拆解。
- 当前项目上下文。
- 调研范围。
- 最终推荐结论。
- 适用边界。

## `search_queries.md`

必须包含：
- 英文关键词。
- 中文关键词。
- GitHub 查询词。
- 官方文档查询词。
- Issues / troubleshooting 查询词。
- 论文查询词。
- 关键词生成逻辑。

## `sources.md`

必须包含：
- 来源标题。
- 链接。
- 来源类型。
- 可信度。
- 相关点。
- 是否推荐继续阅读。
- license / 使用风险。
- 备注。

## `solution_categories.md`

必须包含：
- 方案类别。
- 每类方案核心思想。
- 代表来源。
- 适用场景。
- 不适用场景。

## `solution_comparison.md`

必须包含对比表：
- 方案。
- 优点。
- 缺点。
- 实现成本。
- 维护成本。
- 当前项目适配度。
- 风险。
- 推荐等级。

## `recommended_approach.md`

必须包含：
- 最终推荐方案。
- 推荐理由。
- 取舍说明。
- 为什么不推荐其他方案。
- MVP 实现路径。
- 标准版实现路径。
- 后续增强路线。

## `implementation_notes.md`

必须包含：
- 架构建议。
- 模块划分。
- 数据结构建议。
- API / 接口设计。
- 状态管理建议。
- 配置建议。
- 测试建议。

## `risks_and_tradeoffs.md`

必须包含：
- 技术风险。
- 安全风险。
- license 风险。
- 维护风险。
- 成本风险。
- 用户体验风险。
- 规避建议。

## `codex_tasks.md`

必须包含可直接交给 Codex 的任务：
- 任务标题。
- 目标。
- 涉及文件。
- 实现步骤。
- 验证方式。
- 风险检查点。
- 不要做的事情。

## `open_questions.md`

必须包含：
- 当前无法确认的问题。
- 需要进一步搜索的问题。
- 需要用户决策的问题。
- 需要运行项目确认的问题。

## 深度调研模板

深度调研应包含多关键词、多来源、多方案对比、风险和取舍、完整 `.reference-research/` 文档，以及 Codex 任务。

## 快速调研模板

快速调研只输出：
- 3-5 个关键结论。
- 推荐方向。
- 主要风险。
- 下一步建议。

默认不创建文件，除非用户明确要求。
