# Source Quality Grading

## S 级

标准：
- 官方文档、官方示例、标准文档。
- 论文原文或正式技术报告。
- 主流项目官方仓库。

使用方式：
- 可作为核心依据。
- 仍需检查版本、日期、适用范围和 license。

示例：
- Framework official docs.
- Maintained official examples repository.
- Peer-reviewed paper or official arXiv preprint from the authors.

## A 级

标准：
- 活跃开源项目。
- 高质量技术博客或维护良好的教程。
- 官方团队成员文章。
- 高质量 GitHub Issues / Discussions，尤其有维护者回复。

使用方式：
- 可作为重要参考。
- 需要和 S 级来源交叉验证。

## B 级

标准：
- 普通博客、社区回答、Stack Overflow。
- Reddit / Hacker News 讨论。
- 小型项目 README。

使用方式：
- 适合补充真实经验和坑点。
- 不应单独支撑关键决策。

## C 级

标准：
- 过时文章。
- 不完整 demo。
- 无 license 项目。
- 维护停止项目。
- 来源不清楚的教程。

使用方式：
- 只能作为边缘参考。
- 如果引用，必须标注局限和风险。

## D 级

标准：
- 明显广告。
- 复制粘贴内容。
- 无法验证来源。
- 与需求相关性弱。
- 过时且无说明。

使用方式：
- 默认排除。
- 不用于推荐方案。

## 处理过时和低质量来源

- 检查发布日期、最近更新时间、版本号和依赖版本。
- 对 GitHub 仓库检查最近 commit、issue 活跃度、release、license、examples。
- 对博客和教程检查是否仍匹配当前官方文档。
- 对无 license 仓库，明确提醒不可直接复用代码。
- 如果来源不足，明确说明结论不确定，不要补造证据。
